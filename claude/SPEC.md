# kit_teamfight ゲーム仕様書

Paper 1.21 ベースのチーム対戦プラグイン。革装備+石剣でスタートし、時間経過で貯まるポイントで kit を購入して装備を強化、旗の占拠と敵キルで相手チームのチケットを削り、ゼロにしたら勝利。

---

## 1. 概要

- **ジャンル**: チーム対戦 (拠点占拠 + ティケット消費型)
- **勝利条件**: 自チーム以外の全チームのチケットを 0 にする
- **制限時間**: なし (決着がつくまで継続)
- **初期構成**: 2チーム対戦 (RED / BLUE)。設定でチーム数を増やせる
- **旗**: コマンドで自由に設置/撤去できる。0個でも試合は成立する

---

## 2. チーム (Teams)

### 初期構成
- RED / BLUE の 2チーム
- 各チームは以下を保持:
  - チーム ID (一意)
  - 表示名 / カラー
  - スポーン座標
  - 初期チケット (デフォルト 100)

### 拡張性
- チーム数は `config.yml` の定義で任意に増減可能
- チーム参照は `TeamId` (文字列キー) で抽象化
- チケット減少先決定・勝敗判定はチーム集合をループする実装にし、3チーム以上にも対応する
- 旗の所有者・キャプチャ判定もすべて `TeamId` 経由

### プレイヤー振り分け
- デフォルト: 自動均等 (人数が少ないチームへ自動配属)
- コマンドでの手動選択も用意 (`/team join <teamId>`)

---

## 3. 初期装備 (Starting Loadout)

スポーン / リスポーン時に付与:
- 革のヘルメット
- 革のチェストプレート
- 革のレギンス
- 革のブーツ
- 石の剣

ただし kit を保有している場合は、初期装備ではなく**保有 kit を再適用**する。

---

## 4. ポイントシステム (Point System)

### 基本仕様
- プレイヤー単位でポイントを保持
- 時間経過で自動加算 (デフォルト: 1pt/秒、`config.yml` で調整可)
- 死亡してもポイントはリセットされない (試合終了で初期化)

### 加算ソースの抽象化
加算ソース:
- 一定間隔で加算 (`point.amountPerTick`、実装済み)
- **キル時に加算 (実装済み)**: 敵を1キルすると `point.killBonus` (既定10pt)。`GameManager.onKill()`。
  自殺・味方殺しは加点しない。
- 旗保持中に加算 (将来拡張)

---

## 5. kit システム (Kit System)

枠組みのみ定義する。具体的な kit ラインナップは後続タスクで追加。

### Kit インターフェース (概念)
```
Kit {
  id: String              // 一意識別子
  displayName: String     // 表示名
  cost: int               // 購入に必要なポイント
  apply(Player): void     // 装備一式をプレイヤーに適用 (既存装備を完全置換)
}
```

### 保持ルール
- プレイヤーは**同時に 1つの kit のみ**保持可能
- 別 kit を購入したときの挙動:
  1. プレイヤーの装備 (armor + メインハンド) をすべて剥がす
  2. 新 kit の装備を適用
  3. 保有 kit を新 kit に更新
- リスポーン時は保有 kit を再適用 (kit 未保有なら初期装備)
- 試合終了で保有 kit はリセット

### 購入 UI
- コマンド `/kit shop` でチェスト UI を開く
- 各スロットに kit を配置、購入可能なら緑、ポイント不足なら赤などの視覚表現

### 拡張ポイント
新 kit の追加は `Kit` 実装クラスを 1つ追加 + 登録するだけ。コア側のロジックは変更不要。

## 6. 旗 (Flags)

### 設置方法
旗は**コマンドで動的に設置/撤去**する。マップ依存ではなく運営者が自由に配置できる。

### 旗管理コマンド (案)
- `/flag create <flagId>` — 自分の立ち位置に旗を新規作成 (所有者なし = 中立)
- `/flag remove <flagId>` — 旗を削除
- `/flag list` — 設置済み旗一覧
- `/flag setowner <flagId> <teamId|none>` — 初期所有者を指定 (拠点旗運用のとき便利)

### 旗の個数
- **任意個数で動作する**。0個でも 1個でも N個でも試合は成立する
  - 0個の場合: チケット減少要因は「キル」のみ → ピュアな TDM 寄りになる
  - 多数の場合: 占拠ゲーム寄りに振れる
- 推奨は 3個 (各陣営拠点 + 中央) だが、推奨であって必須ではない

### キャプチャ仕様
- 旗の周囲 (デフォルト半径 5ブロック、調整可) に**現所有者以外**のプレイヤーが入ると、キャプチャ進捗が進む
- 必要時間 (デフォルト 5秒、調整可) 経過で所有者がキャプチャ側チームに切り替わる
- キャプチャ中に現所有者チームのプレイヤーが範囲に入ると進捗が**停止** (巻き戻りはなし — シンプル優先)
- 範囲内に複数チームのプレイヤーがいる場合: いずれのチームもキャプチャ進捗を進められない (膠着)
- 自チーム所有の旗は、他チームに奪われない限り保持される

### 視覚表現
- 設置位置にバナー or ビーコンを配置 (実装時に決定)
- 所有チームのカラーで表示

---

## 7. チケットシステム (Ticket System)

### 初期値
- 各チームは初期チケットを保有 (デフォルト 100、`config.yml` で調整可)

### 減少ルール
1. **キルによる減少**
   - プレイヤーがキルされたとき、そのプレイヤーの所属チームのチケットが -1
   - キル時のチケット減少量も調整可

2. **旗保持による減少**
   - 一定間隔 (デフォルト 5秒) で、旗を所有しているチームから「敵チーム」に対してチケット減少を発生させる
   - 自チームが N 個の旗を所有 → 敵チームのチケットを N pt 減少
   - **旗が 0個の試合では本ルールは無効化** (キル減少のみで進行)
   - **3チーム以上の場合**: 自チーム以外の全敵チームから均等に減らす

### 勝利判定
- いずれかのチームのチケットが 0 (以下) になった瞬間に試合終了
- 残ったチームのうち最大チケット保有チームが勝者
  - 2チーム時: 残ったチームが勝者
  - 3チーム以上で同時にゼロが起きた場合: 直前のチケット数で判定 → 同値なら引き分け

## 8. 調整可能パラメータ (config.yml)

すべての数値は `config.yml` で外出しする。

| カテゴリ | キー | デフォルト | 説明 |
|---|---|---|---|
| Point | `point.tickRateSeconds` | 1 | ポイント加算間隔 (秒) |
| Point | `point.amountPerTick` | 1 | 1回の加算量 |
| Point | `point.killBonus` | 10 | 敵を1キルしたときの獲得ポイント |
| Flag | `flag.captureRadius` | 5 | キャプチャ判定半径 (ブロック) |
| Flag | `flag.captureSeconds` | 5 | キャプチャ完了までの秒数 |
| Flag | `flag.ticketDrainIntervalSeconds` | 15 | 旗保持によるチケット減少間隔 (※コマンド `/ktf flagdrain` で変更可) |
| Flag | `flag.ticketDrainPerFlag` | 1 | 旗 1個あたりの減少量 |
| Ticket | `ticket.initial` | 100 | チームごとの初期チケット |
| Ticket | `ticket.lossPerKill` | 1 | キル時の減少量 |
| Team | `teams[].id` / `name` / `color` / `spawn` | — | チームごとの定義 |

旗の座標は config に書かず、コマンド設置の結果として `flags.yml` のような別ファイルに永続化する。

---

## 9. 拡張性 (Extensibility)

設計時に意識する拡張ポイント:

- **チーム追加**: `config.yml` にチーム定義を追加するだけ。コード変更不要
- **kit 追加**: `Kit` 実装クラスを追加してレジストリに登録するだけ
- **ポイント加算ソース追加**: `PointSource` 実装を追加するだけ
- **旗追加/削除/移動**: 試合中でもコマンドで実施可能 (運用上の柔軟性)

チーム数依存・旗数依存の処理は、すべてコレクションをループする形で書き、N=0 や N=1 のエッジケースで破綻しないこと。

- リスポーン待機時間 (即時 / N秒待機)
- 試合進行の状態管理 (ロビー → 開始 → 終了 → リセット)
- 試合の開始トリガー (コマンド / 人数到達で自動)
- 観戦モードの有無
- スコアボード / ボスバー表示内容 (チケット残量、自分のポイント、保有 kit など)
- 旗の視覚表現の最終決定 (バナー / ビーコン / 両方)
- フレンドリーファイア有無
- 試合終了後の挙動 (ロビーに戻す / 自動再戦)
- 永続化方式 (旗座標のファイル形式、再起動時の復元)
