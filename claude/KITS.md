# Kit 設計メモ

kit ラインナップの仕様。**実装済み**(`src/main/java/lobby/kit_teamfight/kit/kits/` に5クラス)。
新規 kit は `Kit` を実装したクラスを作り、`Kit_teamfight.onEnable()` で `register(...)` するだけ
(コア側の変更は不要)。旧サンプル `ArcherKit`(中身を差し替え)/ `KnightKit`(削除)はこの案で置き換え済み。

---

## 共通仕様

- **全 kit にパン × 2スタック (128個)** を配布する。
- **騎兵系 (cavalry) kit は追加で小麦 × 2スタック** を配布する(馬の回復/管理用)。
- kit は1つだけ保持し、別 kit 購入で**装備を完全置換**(既存の装備・インベントリを clear → 適用)。
- リスポーン時は保有 kit を再適用。死亡で kit はリセットされ初期装備に戻る。
- **ポーション効果は全除去 (確定)**: **どの kit を選んでも(切替時)・および死亡時に、付与中のポーション効果を
  すべて消す**。その上で、選んだ kit が効果を持つ場合(例: スナイパーの鈍足II)だけ付け直す。

### 騎兵系 (cavalry) について

- 一部の kit は「騎兵系」に分類される。**騎兵系 kit のときだけ馬に乗れる**。
  それ以外の kit / 初期装備のプレイヤーは馬に騎乗できない。
- 騎兵系 kit を装備すると、**スポーン時に専用の馬が一緒に出現**する(指定の馬鎧を装備済み)。
- **スポーン時はプレイヤーを自動でその馬に騎乗させる (確定)**。
- **馬は復活しない (確定)**: 殺された/失った馬は再発行しない。騎兵系 kit を選び直すと新しい馬が出る。
- 現在の騎兵系: `lancer`(槍騎兵) / `robinhood`(ロビンフット)。今後追加する馬持ち kit もここに分類する。

## 各 kit の詳細

### アーチャー (archer) — 50pt
- 防具: 革のヘルメット + 革のチェストプレート(**上半身のみ**。脚・足なし)
- 近接: 石の剣
- 弓: **無限 (Infinity)** エンチャント
- 矢: 無限弓は最低1本の矢が必要 → **矢を1本だけ配布**

### 軽散兵 (skirmisher) — 100pt
- 防具: 革のヘルメット + 革のチェストプレート(**上半身のみ**)
- 近接: 石の斧
- 盾: **オフハンド**に1つ
- ※ クロスボウ/矢は廃止(近接寄りの構成に変更)

### 槍騎兵 (lancer) — 180pt 【騎兵系】
- 防具: **チェーン(チェーンメイル)全身**(ヘルメット/チェスト/レギンス/ブーツ)
  - ※ ユーザー記述「前身チェーン」は「全身チェーン」と解釈
- 武器: **鉄の槍** (`Material.IRON_SPEAR`)。※以前はネザライトの槍だったが弱体化
- サブ武器: **石の剣**(接近戦用)
- 馬: スポーン時に**鉄の馬鎧を装備した馬**が一緒に出現。
  - **HP は通常(22)の 3/2 = 33**(以前の 44 から弱体化)
  - **移動速度は通常(0.225)**(2/3に下げたが遅すぎたため通常へ戻した)、ジャンプは平均(0.7)
- 小麦: 2スタック(騎兵系共通)

### ロビンフット / 弓騎兵 (robinhood) — 125pt 【騎兵系】
- 防具: **革(全身)**(ヘルメット/チェスト/レギンス/ブーツ)
  - ※ ユーザー記述「前身革」は「全身革」と解釈
- 近接: 石の剣
- 弓: **無限 (Infinity)** エンチャント
- 矢: 無限弓用に1本配布
- 馬: スポーン時に**革の馬鎧を装備した馬**が一緒に出現
- 小麦: 2スタック(騎兵系共通)

### 軍兵 (soldier) — 150pt
- 防具: **鉄装備フルセット**(ヘルメット/チェスト/レギンス/ブーツ)
- 近接: **ダイヤの剣**(主武器) + **木の斧**(盾割り用)
- 飛び道具: **クロスボウ × 1**(耐久Iの見た目エンチャントのみ・貫通なし) + 矢3スタック (192本)

### 重弩兵 (arbalest) — 110pt
- 防具: **革装備一式**(ヘルメット/チェスト/レギンス/ブーツ)
- 近接: **石の剣**(接近されたとき用)
- 飛び道具: **貫通 (Piercing IV) クロスボウ × 3**(装填の遅さを3丁の持ち替えで補うバースト型)
- 矢: 4スタック (256本)
- 効果: **常時 鈍足 I**(移動速度低下 I)

### スナイパー (sniper) — 240pt
- 防具: なし
- 近接: なし(剣を渡さない)
- 弓: **射撃ダメージ増加 (Power) III + 無限 (Infinity)**
- 効果: **常時 鈍足 II**(移動速度低下 II)を付与
- 矢: 無限弓用に1本配布

## 実装前に詰める論点 (要確認)

1. **「ネザライトの槍」(解決)**: 1.21.11 では槍が実在する → `Material.NETHERITE_SPEAR` をそのまま使う。
   (Material には `WOODEN`/`STONE`/`COPPER`/`IRON`/`GOLDEN`/`DIAMOND`/`NETHERITE_SPEAR`、`MACE`、`TRIDENT` あり)

2. **ポーション効果の扱い (確定)**: `clearEquipment()`(装備とインベントリのみ)では効果は消えない。
   - **方針**: **kit を選ぶたび(切替時)と死亡時に、付与中の全ポーション効果を除去**する。
     その後、選んだ kit が効果持ち(スナイパー=鈍足II)なら付け直す。スナイパーは無限時間の鈍足IIを付与し、
     リスポーン/再適用のたびに付け直す。
   - 実装場所: `equip()`/`buyKit()` の冒頭で `player.getActivePotionEffects()` を全 `removePotionEffect`、
     `PlayerListener.onDeath` でも除去。kit 側は `apply()` 内で必要な効果だけ付ける。

3. **無限弓の矢**: 無限弓は矢が1本でもインベントリに無いと撃てない。→ アーチャーに矢1本を必ず配る(上記反映済み)。

4. **スナイパーの矢 (確定)**: 弓に**無限**が付くので矢は1本だけ配布(無限弓と同様)。

5. **「上半身のみ」の解釈**: ヘルメット + チェストプレート と解釈(レギンス・ブーツなし)。これで良いか確認。

6. **盾・クロスボウのスロット**: 盾はオフハンド、クロスボウはホットバー想定。パン2スタックと合わせて
   ホットバー配置(並び順)を決めておくと親切。

7. **コストバランス**: 50 / 100 / 180 / 240 の順。現状の `point.*`(時間経過レート)との兼ね合いで、
   各 kit に到達するまでの体感時間を後で調整する。

8. **既存サンプル kit の扱い**: 今の `ArcherKit`(50pt)/`KnightKit`(60pt)はこの案で差し替える想定。
   `archer` は名前が被るので、サンプルを削除してから新 `archer` を登録する。

9. **騎兵系 (cavalry) の分類**: `Kit` に「騎兵かどうか」を表すフラグ(例: `isCavalry()`)を持たせ、
   馬の騎乗可否・馬の出現・小麦配布をそれで分岐する。

10. **馬の出現と性能 (実装済み)**: `GameManager.spawnCavalryHorse()` が馬を1頭スポーン。
    - 飼い慣らし済み(owner=本人)・サドル付き・指定の馬鎧(lancer=鉄 / robinhood=革、BODY スロット)。
    - **性能固定値**: 既定は体力 **22.0HP**、移動速度 **0.225**、ジャンプ **0.7**
      (`Attribute.MAX_HEALTH / MOVEMENT_SPEED / JUMP_STRENGTH`)。
      HP は `Kit.horseMaxHealth()`、速度は `Kit.horseMovementSpeed()` で kit ごとに上書き可能
      (lancer は HP 33。速度は既定の 0.225)。
    - **自動騎乗**: `horse.addPassenger(player)`。`setPersistent(false)` で再起動時に残さない。

11. **馬の搭乗制限 (実装済み)**: `PlayerListener.onMount`(`EntityMountEvent`)で、
    **騎兵系 kit でないプレイヤーが馬(AbstractHorse)に乗るのをキャンセル**する。
    - **敵の馬でも騎兵系なら乗れる (確定)**: 馬の所有者は問わず、騎乗者が騎兵系かどうかだけで判定。
    - ボート/トロッコ等は対象外。

12. **馬のライフサイクル(重要)**: 馬が増殖しないよう、プレイヤーごとに出した馬を追跡して掃除する。
    - kit 切り替え時・死亡時・リスポーン時・退出時に**古い馬を除去してから**新しい馬を出す。
    - **馬は復活しない (確定)**: 馬が殺された/見失っても**再発行しない**。新しい馬が欲しければ
      騎兵系 kit を選び直す(= kit を購入し直してスポーンし直す)必要がある。
    - 補足: 死亡すると kit はリセットされ初期装備に戻る(騎兵でなくなる)ため、リスポーン時に馬は出ない。
      騎兵系 kit を再購入したときだけ新しい馬が1頭出る。
    - `Map<UUID, 馬のUUID>` 的な管理 + クリーンアップ処理が必要。

13. **小麦の用途**: 騎兵系は小麦2スタック。馬の回復/管理用という位置づけ(パン2スタックとは別枠で追加)。

## 実装時の段取り (メモ)

- `kit/kits/` に `ArcherKit` / `SkirmisherKit` / `LancerKit` / `RobinhoodKit` / `SniperKit` を作成。
- 各 `apply(Player)` で防具・武器・矢・パンを配布。スナイパーは鈍足IIも付与。騎兵系は小麦+馬も。
- `Kit` に騎兵フラグ(`isCavalry()`)を追加し、馬出現・搭乗制限・小麦配布をそれで分岐(論点9〜13)。
- ポーション効果を使う kit があるため、`clearEquipment()` か `buyKit()`/`equip()` に
  **「既存のポーション効果を消す」処理**を足すかを設計時に決める(論点2)。
- **馬の管理**: プレイヤーごとに出した馬を追跡 (`Map<UUID, UUID>` 等) し、
  kit切替/死亡/リスポーン/退出で古い馬を除去 → 新規スポーン(論点12)。
- **搭乗制限リスナー**: `EntityMountEvent` 等で騎兵系以外の騎乗をキャンセル(論点11)。
- 無限/射撃増加/盾/クロスボウ/馬鎧/属性固定など、生成ヘルパーがあると重複が減る。
